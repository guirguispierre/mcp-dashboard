üìñ Integration Types Explained

Understanding the different types of MCP integrations

1

Built-in Poke Integrations

These integrations are native to Poke.com and require zero server setup. Simply connect your account in Poke settings and start using them immediately. Examples: GitHub, Todoist, Asana, Linear, Notion, Supabase, Vercel, Netlify, DeepWiki, Sentry, Ramp, and Webflow.

2

Custom MCP Servers

These are community-created MCPs deployed on Render by @guirguispierre. They require adding the server URL to your Poke MCP configuration. Examples: BART, PDF Reader, Last.fm, FFmpeg, and Earnings Feed.

3

Configure Custom MCPs

For custom MCPs, you'll need to add the server URL to your Poke settings. Each integration card shows the specific server URL. Some may require additional API keys (like BART).

4

Start Using

Once configured, simply ask Poke naturally! For example: "Show my GitHub repos" or "What's the next BART train?" The integrations work seamlessly in your conversations.

‚ùì Frequently Asked Questions

Everything you need to know about MCP integrations

What's the difference between built-in and custom MCPs? ‚ñº

Built-in integrations are native to Poke.com and require no server setup‚Äîjust connect your account. Custom MCPs are community-created servers that need to be configured in your Poke settings with a server URL. Built-in integrations are more convenient, while custom MCPs offer specialized functionality.

How do I add a custom MCP server to Poke? ‚ñº

Go to your Poke settings, find the MCP configuration section, and add the server URL provided on each custom integration card. Some integrations may require additional API keys. Click "View Guide" on any custom integration for detailed setup instructions.

Do custom MCPs hosted on Render cost money? ‚ñº

The custom MCP servers themselves are free to use. However, Render's free tier may have limitations. If you experience issues, you might need to deploy your own instance or wait for the server to wake up from sleep (common with free-tier hosting).

Why does BART require an API key? ‚ñº

BART's real-time transit data API requires authentication. You need to register for a free API key at api.bart.gov to access their data. This ensures fair usage and helps BART track API utilization.

Can I create my own custom MCP? ‚ñº

Absolutely! Check out the Model Context Protocol documentation at modelcontextprotocol.io to learn how to build your own MCP server. You can deploy it on Render, Railway, or any hosting service and connect it to Poke.

Is my data secure with custom MCPs? ‚ñº

Custom MCPs act as intermediaries between Poke and external services. Always review the source code (all custom MCPs here are open source on GitHub) and use API keys with minimal required permissions. Built-in Poke integrations follow Poke's security standards.
